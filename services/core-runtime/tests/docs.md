# tests/ — Test Pyramid

The test suite follows a pyramid structure: many fast unit tests at the base, fewer integration tests in the middle, and targeted contract and replay tests at the top.

## Role in the System

Tests verify every layer of the architecture:
- **Unit tests** validate individual classes and methods in isolation
- **Integration tests** verify the full `EditEngine` pipeline
- **Contract tests** ensure JSON output matches the defined schemas
- **Replay tests** verify deterministic behavior across independent runs

## Running Tests

```bash
pytest -q                    # all tests
pytest -q tests/unit         # unit only
pytest -q tests/integration  # integration only
pytest -q tests/contract     # contract only
pytest -q tests/replay       # replay only
```

Or via Docker:
```bash
docker compose run --rm test
docker compose run --rm contract-test
docker compose run --rm replay-test
```

### `conftest.py` — Path Setup

Adds the project root to `sys.path` so that `import core.xxx` works from within the tests directory. This is necessary because the project is not installed as a package during development.
